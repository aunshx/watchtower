================================
PART 2: EXPANDED CARD VIEW REDESIGN
================================

When a user clicks a pain moment card, it expands to show the artifacts. Make the expanded view feel like a polished detail view, not a cramped panel.

Current problems to fix:
- Cluttered: too much info crammed together
- Tab states are unclear
- Markdown renders without spacing/hierarchy
- Section boundaries aren't clear

Required redesign:

When expanded, the card section should have this structure:

1. The collapsed card preview stays at top (rank, score, chip, repo, signal) — same as now
2. Below it, a horizontal divider (border-t border-stone-200, my-6)
3. A "Source" line: github icon + "Source: " label in text-stone-500, then the full URL as a link in text-blue-600 (truncate gracefully if too long, but show ellipsis at the end)
4. Below source, the tab bar:
   - Three tabs: "Public comment" / "Outreach" / "Case study"
   - Tabs are flex items with px-4 py-3 spacing
   - Inactive: text-stone-600 hover:text-stone-900
   - Active: text-blue-600 font-medium with border-b-2 border-blue-600 underneath
   - Inactive tabs have no bottom border (or transparent)
   - Add the artifact word count or length next to each tab name in a small text-stone-500 caption (e.g., "Public comment · 87 words")
5. Below the tabs, the active artifact content with these styles:
   - Container: bg-stone-50 rounded-lg p-6 mt-4 border border-stone-200
   - Use react-markdown's prose styling. Add Tailwind Typography classes: prose prose-stone max-w-none
   - If Tailwind Typography isn't installed: npm install -D @tailwindcss/typography and add it to tailwind.config.js plugins
6. Below the artifact content, a small footer line: "Draft only — not posted to GitHub or sent." in text-stone-500 text-sm italic, with appropriate top spacing
